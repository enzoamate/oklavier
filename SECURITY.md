# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ |
| < 1.0   | ❌ |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Please report privately via GitHub:

1. Go to https://github.com/enzoamate/oklavier/security/advisories/new
2. Or email **eamate@oklavier.com** with the details

Include:
- A description of the vulnerability
- Steps to reproduce (or a proof-of-concept)
- Affected versions
- Potential impact

You will receive an acknowledgement within **72 hours**, a fix timeline within **7 days**, and credit in the security advisory once the fix is released (unless you prefer to remain anonymous).

## Disclosure Policy

We follow coordinated disclosure: the vulnerability is kept private until a patched release is available, then published as a [GitHub Security Advisory](https://github.com/enzoamate/oklavier/security/advisories) with a CVE if applicable.
